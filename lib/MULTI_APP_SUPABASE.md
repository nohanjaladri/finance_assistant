# Panduan: Multi-Application Supabase dengan Schema Terpisah

Dokumen ini menjelaskan cara kerja arsitektur multi-schema database pada project "Dompetku AI", cara menambah schema baru, serta panduan pengembangannya.

---

## 🏗️ Desain Arsitektur

Secara default, Supabase menaruh tabel di skema `public`. Untuk mendukung multi-app dalam satu database, kita memindahkan seluruh objek aplikasi ke skema PostgreSQL tersendiri (contoh: `app_finance`).

- **Satu Supabase Project:** Membagi resource database secara logis menggunakan skema PostgreSQL.
- **Isolasi Data:** Setiap aplikasi terisolasi pada skema masing-masing, sehingga data tidak saling mengintervensi.
- **Konfigurasi Terpusat:** Nama skema didefinisikan secara dinamis pada file `.env` di baris `APP_SCHEMA=app_finance`.

---

## 🚀 Panduan Pengembangan

### 1. Cara Mengubah Schema Aktif (`APP_SCHEMA`)
Cukup edit nilai variabel `APP_SCHEMA` di file `.env` pada root directory aplikasi:
```env
APP_SCHEMA=app_pos  # Ganti dengan skema aplikasi Anda
```
Aplikasi akan secara otomatis memetakan seluruh query data, RPC, dan realtime listener ke skema kustom tersebut.

### 2. Cara Menambahkan Schema Baru
1. Salin script migrasi [supabase_schema.sql](file:///d:/Data/Project_Antigravity/finance_assistant/supabase_schema.sql).
2. Ganti semua string `app_finance` menjadi nama skema baru Anda (misalnya `app_inventory`).
3. Jalankan script SQL tersebut di **Supabase Dashboard -> SQL Editor**.

### 3. Cara Menambahkan Tabel Baru pada Schema Terpisah
Tulis perintah DDL SQL dengan menyertakan prefix skema aktif di awal nama tabel:
```sql
CREATE TABLE app_finance.kategori_baru (
  id BIGSERIAL PRIMARY KEY,
  name TEXT NOT NULL
);

-- Jangan lupa aktifkan RLS
ALTER TABLE app_finance.kategori_baru ENABLE ROW LEVEL SECURITY;
```

## 🖥️ Dampak terhadap Supabase Studio
- **Database Schema Dropdown:** Di Supabase Studio (Table Editor), Anda tidak akan melihat tabel di skema `public`. Anda harus mengklik dropdown schema di pojok kiri atas table editor dan menggantinya ke skema aktif Anda (misalnya `app_finance`).
- **Autogenerated API Docs:** API documentation Supabase akan menyesuaikan secara otomatis untuk skema yang dipilih.
